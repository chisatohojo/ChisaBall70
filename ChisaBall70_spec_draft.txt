ChisaBall 70 仮 仕様書ドラフト
作成日: 2026-05-23

============================================================
1. コンセプト
============================================================

US配列準拠の左右分割・平面エルゴキーボード。
右手側にトラックボールを搭載し、Bluetooth接続のキーボード＋マウス複合HIDとして動作させる。

完全ワイヤレスではなく、PCとはBluetooth接続、左右間は有線接続とする。
バッテリーは右手側のみ搭載し、左手側は右手側から給電する。

キーキャップ入手性を保ちつつ、親指操作・トラックボール・分割エルゴを組み合わせた
実用寄りの変なキーボードを作ることを目的とする。


============================================================
2. 全体構成
============================================================

PC
  ↑ Bluetooth
右手側
  ・Seeed Studio XIAO nRF52840
  ・LiPoバッテリー
  ・MCP23017 右手キー用
  ・MX8650A トラックボールセンサー
  ・マウスボタン類
  ↓ 有線 I2C + 給電
左手側
  ・MCP23017 左手キー用
  ・キースイッチ群
  ・マイコンなし
  ・バッテリーなし

PCから見えるデバイス:
  ・Bluetoothキーボード
  ・Bluetoothマウス
  ・可能なら複合HIDとして1デバイス扱い


============================================================
3. 右手側構成
============================================================

メインマイコン:
  ・Seeed Studio XIAO nRF52840 通常版
  ・XIAO nRF52840 Plusは入手困難なため使わない方針

電源:
  ・右手側のみLiPo搭載
  ・右手側USB-Cで充電
  ・右手側に電源スイッチを配置
  ・左手側へ3.3V/GNDを供給

入力:
  ・右手キーは右手MCP23017で読み取り
  ・左手キーは左手MCP23017をI2C越しに読み取り
  ・トラックボールはMX8650AをXIAOから直接読み取る

通信:
  ・PCとはBluetooth
  ・左手とは有線I2C


============================================================
4. 左手側構成
============================================================

構成:
  ・MCP23017
  ・キースイッチ
  ・ダイオード
  ・左右接続コネクタ

搭載しないもの:
  ・マイコン
  ・LiPoバッテリー
  ・充電回路
  ・USB接続機能

左手側は「キーがたくさん付いたI/O箱」として扱う。


============================================================
5. 左右間ケーブル
============================================================

基本4線:
  ・3V3
  ・GND
  ・SDA
  ・SCL

必要に応じて追加:
  ・INT

コネクタ候補:
  ・JST-SH
  ・JST-GH
  ・USB-Cコネクタを独自配線で使用

優先度低め:
  ・TRRS
    理由: 通電中の抜き差し事故が怖い

運用ルール:
  ・左右ケーブルの抜き差しは電源OFF時に行う
  ・充電は右手側USB-Cのみ
  ・左手側にはUSB端子を搭載しない、または通常運用で使わない


============================================================
6. 電源構成
============================================================

バッテリー:
  ・右手側のみLiPo

給電:
  ・右手XIAO nRF52840周辺の3.3V系から以下へ給電
    - 右手MCP23017
    - 左手MCP23017
    - MX8650Aセンサー
    - 必要なLED等

注意:
  ・初期版ではOLEDやRGB LEDなどの消費電力部品は搭載しない
  ・右手側レギュレータの電流余裕を確認する
  ・MX8650AとLED周辺の消費電流を実測したい
  ・左手側へ給電するため、左右ケーブルのGNDは確実に接続する


============================================================
7. GPIOエキスパンダ
============================================================

使用IC:
  ・MCP23017

数量:
  ・右手側 1個
  ・左手側 1個

通信:
  ・I2C

アドレス例:
  ・右手 MCP23017: 0x20
  ・左手 MCP23017: 0x21

用途:
  ・片側5行×7列のキーマトリクス読み取り
  ・クリックキー、スクロール切替キーも可能なら行列内に含める

ピン割り当て案:
  ・GPA0〜GPA4: Row 0〜4
  ・GPB0〜GPB6: Col 0〜6
  ・GPA5/GPA6など: 予備
  ・GPA7/GPB7: 入力列には使わない方針。必要なら出力用途または未使用

キースイッチ:
  ・各キーにダイオードを付ける
  ・ゴースト防止、同時押し安定化のため

ZMK上の扱い:
  ・ZephyrのMCP23017 GPIOドライバを利用する想定
  ・ZMKのmatrix kscanにMCP23017 GPIOを渡せるか実験する
  ・まず1個のMCP23017で3×3程度の小型マトリクスを確認してから5×7へ拡張する


============================================================
8. トラックボール
============================================================

流用元:
  ・ダイソー無線マウス基板

センサー:
  ・MX8650A

参考:
  ・以前使用経験あり
  ・ftukurou.blog41.fc2.com のダイソーマウス記事
  ・zapmaker/OptiMouse
  ・MX8650系Arduinoライブラリ

接続予定:
  ・VDD
  ・GND
  ・SCLK
  ・SDIO
  ・必要ならCS / MOTION / LED制御

方針:
  ・まずArduino環境でX/Y deltaが読めるか確認
  ・その後、ZMK/Zephyr向けに読み取り処理を移植
  ・ZMKではBluetoothマウス移動として送信する
  ・Scrlキー押下中はトラックボール移動をスクロールとして扱う

機構注意:
  ・レンズとボール表面の距離が重要
  ・元マウスの光学距離をなるべく維持する
  ・センサー、レンズ、LED、周辺部品はなるべくセットで流用する
  ・透明、白一色、ツルツル過ぎる球は避ける
  ・ボール支持機構は別途検討する

============================================================

本命:

ZMKで担当すること:

  ・Scrlキー押下中のスクロール変換

開発順:

  4. MCP23017 2個を同一I2Cバスで読み取り確認
  5. 左右ケーブル越しに左MCP23017を読めるか確認
  6. XIAO nRF52840をArduino環境でMX8650A読み取り確認

============================================================
11. キー配列方針
============================================================

基本:
  ・英語キーボード、US配列準拠
  ・キーキャップを探しやすくする
  ・アルファベット、数字、記号はなるべくUS準拠
  ・変えるのは親指キー、Fn、矢印、クリック、スクロール周り

IME:
  ・左手小指側、Caps位置相当にIME切替キーを置く

矢印:
  ・右手下段に配置
  ・↑は右手Shiftの左付近に置く方針

Enter / Backspace:
  ・右上端にBackspace
  ・右中段端にEnter
  ・親指クラスタには無理に入れない

Space:
  ・左親指メイン
  ・右手はトラックボールとクリック操作があるため、Spaceは左優先

13. 親指クラスタ / マウスボタン配置
============================================================

本命案:

左手親指クラスタ

        [Btn1]
[Alt]   [Fn]   [Space]


右手親指クラスタ

        [MBtn]
[Scrl]  [Btn1] [Btn2]

右手内側下

        [Trackball]


役割:

Btn1:
  ・左クリック
  ・左右両方に置く
  ・右手Btn1は通常クリック用
  ・左手Btn1はドラッグ補助用

Btn2:
  ・右クリック
  ・右親指側

MBtn:
  ・中クリック
  ・右親指上側
  ・リンク新規タブ、ホイールクリック用途

Scrl:
  ・押している間だけトラックボール移動をスクロール化
  ・上下移動で縦スクロール
  ・左右移動で横スクロール

Fn:
  ・レイヤー切替
  ・左親指中央付近

Space:
  ・左親指メイン
  ・押しやすい位置に配置

Alt:
  ・左Alt
  ・必要なら通常レイヤーでAlt、別レイヤーで @ / ` なども検討


配置思想:
  ・文字入力中は左親指がSpace/Fn担当
  ・ポインタ操作中は右手がTrackball/Btn1/Btn2/MBtn/Scrl担当
  ・ドラッグ時だけ左手Btn1で押しっぱなし補助をする


============================================================
14. Fn Layer 案
============================================================

ファンクションキー:
  ・1〜0、-_、Bksp周辺をF1〜F12に割り当て

右手矢印:
  ・Fn + ↑ = PageUp
  ・Fn + ↓ = PageDown
  ・Fn + ← = Home
  ・Fn + → = End

マウス:
  ・Fn + Btn1 = ドラッグロック
  ・Fn + Btn2 = 戻る
  ・Fn + MBtn = 進む
  ・Fn + Scrl = 低速カーソルモード

スクロール:
  ・通常のScrl押下中はトラックボールをスクロール化

17. 現時点の結論
============================================================

本命構成:
  ・右手だけ XIAO nRF52840
  ・右手だけ LiPo
  ・左右に MCP23017 を1個ずつ
  ・左右接続は 3V3 / GND / SDA / SCL
  ・トラックボールはダイソーマウス由来MX8650Aを流用
  ・ファームはZMK
  ・US配列準拠の片側5×7平面エルゴ
  ・左手Btn1でドラッグ補助
  ・右手Btn1/Btn2/MBtn/Scrlでマウス操作
  ・Spaceは左親指メイン

設計思想:
  右手XIAOが脳みそ。
  左右MCP23017が手足。
  MX8650Aが目。
  左手は電池なしの静かな半身。
